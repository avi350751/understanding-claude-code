# Claude Crash Course - Personal Project

Welcome to a practical crash course on Claude AI and how to work effectively with Claude as your coding assistant!

## 📚 About This Project

This is a hands-on project designed to demonstrate:
- How to use Claude for coding assistance and project development
- Best practices for AI-assisted development workflows
- Creating and maintaining Python projects with modern tooling

## 📂 Project Structure

- **`main.py`** - Entry point script demonstrating basic program structure and print statements
- **`play_sound.py`** - Utility module that plays Windows audio files using the `winsound` library
- **`pyproject.toml`** - Project configuration file using modern Python packaging standards
- **`README.md`** - This file; project documentation

## 🚀 Getting Started

### Prerequisites
- Python 3.12 or higher
- Windows OS (for audio playback functionality)

### Running the Project

Run the main program:
```bash
python main.py
```

Play an audio file (requires `voice_note.wav` in the project directory):
```bash
python play_sound.py
```

## 🎯 Learning Goals

This project covers:
1. **Project Setup** - Using `pyproject.toml` for modern Python project configuration
2. **Modular Code** - Organizing code into separate modules (`main.py`, `play_sound.py`)
3. **Error Handling** - Graceful handling of missing files and runtime errors
4. **Platform-Specific Features** - Working with Windows audio APIs
5. **AI-Assisted Development** - Leveraging Claude to scaffold, debug, and improve code

## 💡 Key Takeaways for Claude Usage

- Use Claude to understand existing codebases and architecture
- Ask Claude to generate boilerplate and scaffolding code
- Request code reviews and refactoring suggestions
- Get help with debugging and error resolution
